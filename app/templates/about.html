About Stroke Prediction System

Project Overview

The Stroke Prediction System is a secure web application designed for healthcare institutions to manage patient data related to stroke prediction and analysis.

Key Features

  • Secure User Authentication
  • Patient Data Management (CRUD)
  • Dual Database Architecture
  • Input Validation
  • CSRF Protection
  • Password Hashing
  • Responsive UI Design
  • Comprehensive Logging

Technology Stack

Backend
  • Flask 3.0.0
  • SQLAlchemy
  • PyMongo
  • Flask-Login
  • Flask-WTF
Frontend
  • Bootstrap 5
  • Font Awesome
  • Jinja2
  • Custom CSS
Database
  • SQLite (Authentication)
  • MongoDB (Patient Data)

Security Features

Enterprise-Level Security

This application implements multiple layers of security:

  • PBKDF2-SHA256 password hashing
  • CSRF protection on all forms
  • Input validation and sanitization
  • SQL/NoSQL injection prevention
  • XSS prevention through auto-escaping
  • Secure session management
  • Security headers (X-Frame-Options, X-XSS-Protection, etc.)
  • Comprehensive logging without sensitive data exposure

Dataset Information

This system manages data from the Stroke Prediction Dataset, which includes patient demographics, medical history, lifestyle factors, and health metrics to support stroke risk analysis.

Demographics
  • Gender
  • Age
  • Marital Status
  • Work Type
  • Residence Type
Health Metrics
  • Hypertension Status
  • Heart Disease Status
  • Average Glucose Level
  • BMI
  • Smoking Status
  • Stroke Occurrence

Built as a university assessment project demonstrating secure software development practices
